Azure magma - a NSG compiler


What is a Magma?

A magma is one of the simplest structure one can imagine: it's a set equipped with an operation. We use it to store Azure Network Security Groups (NSGs) of a whole organization to reason about their security at scale and automatically. The operation attached to NSGs is also quite simple: it's the union of rules, that we note as a plus sign '+'.
In practice, we have two different magmas: one for the inbound direction, and another one for the outbound one.


Our magmas are always partitioned into two isolated zones (called equivalence classes):

  • - allowed rules
  • - blocked rules
This makes sense, because a security rule cannot be allowed and blocked at the same time. The resulting structure is called a Magma Quotient. This is the actual space in which all the reasoning takes place.


An interesting property of the (otherwise boring) Magma Quotient is that the '+' operation is congruent. It means that whenever we 'add' two security rules together, provided they belong to the same equivalence class, the resulting object remains in the same equivalence class.
